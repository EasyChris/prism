# 产品需求文档 (PRD)：Claude Code Proxy Hub

| 项目名称 | Claude Code Proxy Hub |
| :--- | :--- |
| **平台** | macOS (基于 Tauriv2 开发) |
| **核心定位** | 专为 Claude Code 打造的动态路由网关与流量审计工具 |

## 1. 产品概述

### 1.1 目标
构建一个运行在 macOS 上的轻量级桌面应用，作为 Claude Code 的**本地中转站**。实现“一次配置，随意切换”的无缝体验，并提供完整的流量去向记录与成本审计。

---

## 2. 核心用户流程 (User Flow)
1.  **启动与配置**：用户打开应用，在界面中预设多个配置方案（如：官方-Sonnet、OpenRouter-Haiku）。
2.  **连接**：用户仅需一次性将 Claude Code 的 `BASE_URL` 指向本应用（Localhost）。
3.  **开发与切换**：
    *   用户在终端正常使用 Claude Code。
    *   需更换模型时，**无需重启终端工具**，直接点击 Mac 菜单栏图标切换预设。
    *   下一次回车发送请求时，流量即刻流向新的供应商。
4.  **审计**：用户查看应用内的日志面板，核对刚才的对话走了哪条线路，消耗了多少 Token。

## 技术要求

* tarui2 最新版本 / TypeScript
* react / tailwindcss / Shadcn UI / Radix UI (打造贴近 macOS 的原生质感)
* 状态管理: Pinia / Zustand
* 后端 rust
* 本地存储: Tauri 内置的 Store 插件，用于持久化存储 API Keys 和配置。

## 架构示意图

[ Claude Code (CLI) ] <--(HTTP/SSE)--> [ Tauri App (Localhost) ] <--(HTTPS)--> [ Real API (Anthropic/OpenRouter) ]


## 使用任务管理 Skills

项目已配置 6 个任务管理 skills，可以通过以下命令使用：

- `/task-create [任务名称]` - 创建新任务
- `/task-work [任务序号]` - 执行任务
- `/task-done [任务序号]` - 完成任务
- `/task-list [日期]` - 列出任务
- `/task-search <关键词>` - 搜索任务
- `/task-archive [选项]` - 归档任务

详细使用说明请参考各 skill 的 SKILL.md 文件。
